## 🤝 Contributing to Documentation

When adding new documentation:
1. Place it in the appropriate subject folder
2. Update this README.md with a link
3. Use clear, descriptive filenames
4. Include a table of contents for long documents
5. Keep documentation up-to-date with code changes

Completed phase documentation (Phase 5, 6, 7 records, etc.) should be moved to [`archive/phases/`](archive/phases/).

## 📝 Documentation Standards

- **Format**: Markdown (.md)
- **Style**: Clear, concise, well-structured
- **Examples**: Include code examples where applicable
- **Updates**: Keep in sync with code changes
- **Links**: Use relative links for internal references
