## 2. Coordinate System & Units

### Blender Authoring Space

| Axis | Direction            | Semantic                   |
|------|----------------------|----------------------------|
| X    | Left / Right         | Port (+X) / Starboard (−X) |
| Y    | Front / Back         | Fore (+Y) / Aft (−Y)       |
| Z    | Up / Down            | Dorsal (+Z) / Ventral (−Z) |

**1 Blender unit = 1 metre.** Ship front faces **+Y**. Engines face **−Y**.

### Engine Space (OBJ Export)

| Export Setting  | Value        |
|-----------------|--------------|
| `forward_axis`  | `NEGATIVE_Z` |
| `up_axis`       | `Y`          |

After export: ship nose points along **−Z**, up is **+Y**.

| Ship Class     | Grid Size (m) |
|----------------|---------------|
| SHUTTLE        | 0.5           |
| FIGHTER        | 1.0           |

## 3. Ship Mesh Assets

### What the Plugin Must Generate

For each ship (identified by `{ship_id}` from the JSON definitions):

| Output File | Path | Description |
|-------------|------|-------------|
| Hull mesh | `data/ships/obj_models/{ship_id}.obj` | Complete ship exterior geometry |
| Materials | `data/ships/obj_models/{ship_id}.mtl` | OBJ material library |
| Ship DNA | `data/ship_dna/{ship_class}_{seed}.json` | Brick layout + config (see §11) |
| Albedo map | `data/ships/textures/{ship_id}/{ship_id}_albedo.png` | Base colour |
| Normal map | `data/ships/textures/{ship_id}/{ship_id}_normal.png` | Surface detail |
| Roughness | `data/ships/textures/{ship_id}/{ship_id}_roughness.png` | PBR roughness |
| Metallic | `data/ships/textures/{ship_id}/{ship_id}_metallic.png` | PBR metallic |
| Emissive | `data/ships/textures/{ship_id}/{ship_id}_emissive.png` | Glow regions |
| AO | `data/ships/textures/{ship_id}/{ship_id}_ao.png` | Ambient occlusion |

### Ship Hull Components (per ship)

| Component | Mesh Part Name | Notes |
|-----------|----------------|-------|
| Main hull / spine | `Hull` | Core fuselage body |
| Cockpit / bridge | `Cockpit` or `Bridge` | Command module |
| Wings (small ships) | `Wing_Left`, `Wing_Right` | Fighter/Corvette only |
| Engines | `Engine_L{N}`, `Engine_R{N}` | Main thrust + maneuver |
| Weapon hardpoints | `Weapon_Hardpoint_{N}` | Up to 10 per ship |
| Turrets | `Turret_Hardpoint_{N}` | Base + Ring + Barrel |
| Module slots | `Module_{N}` | Cargo, shields, sensors, etc. |

### Spine Hull Types

| Spine Type | Silhouette | Typical Use |
|-----------|------------|-------------|
| Needle | Long and narrow | Frigates, Interceptors |
| Wedge | Broad bow, narrow aft | Destroyers |
| Hammer | Wide midships block | Cruisers, Battlecruisers |
| Slab | Flat, wide profile | Carriers, Industrials |
| Ring | Toroidal station-like | Capital ships, Titans |

### Generation Pipeline (9 Stages)

```
1. Core hull (spine)
2. Cockpit / bridge
3. Major structures (wings)
4. Engines (archetype-varied)
5. Weapons & turrets
6. Detail modules
7. Interior (optional)
8. Hull taper / deform pass
9. Bevel + auto-smooth cleanup pass
```

| Medium | 1.5× | Destroyer–Cruiser |

| Type | Description |
|------|-------------|
| Projectile | Kinetic, ballistic |
| Energy | Beam weapons |
| Missile | Guided launchers |
| Hybrid | Energy + kinetic mix |
| Mining | Ore extraction |

### Per-Turret Output

| Output File | Path |
|-------------|------|
| Turret mesh | `data/turrets/obj_models/turret_{size}_{type}.obj` |
| Materials | `data/turrets/obj_models/turret_{size}_{type}.mtl` |
| Textures | `data/turrets/textures/turret_{size}_{type}/` |

### Turret Geometry Structure

Each turret consists of three child meshes:

1. **Base** — flat cylinder at the mounting point
2. **Rotation ring** — torus around the base
3. **Barrel** — cylinder extending forward from the ring

### Turret Custom Properties (for engine parsing)

| Property | Type | Example | Description |
|----------|------|---------|-------------|
| `turret_index` | int | `1` | 1-based position index |
| `turret_type` | string | `"projectile"` | Weapon type |
| `tracking_speed` | float | `30.0` | Rotation speed (deg/s) |
| `rotation_limits` | string | `"yaw:360,pitch:90"` | Rotation constraints |
| `hardpoint_size` | float | `1.44` | Visual size of turret |

## 18. Blender Object Hierarchy Convention

When exporting from Blender, objects should follow this naming hierarchy:

```
[PREFIX_]Spaceship_CLASS_SEED (Collection)
├── [PREFIX_]Hull (Parent Object)
│   ├── [PREFIX_]Cockpit
│   ├── [PREFIX_]Bridge
│   ├── [PREFIX_]Engine_L1 / Engine_R1
│   ├── [PREFIX_]Wing_Left / Wing_Right
│   ├── [PREFIX_]Weapon_Hardpoint_1 .. N
│   ├── [PREFIX_]Turret_Hardpoint_1 .. N
│   │   ├── [PREFIX_]Turret_Ring_N
│   │   └── [PREFIX_]Turret_Barrel_N
│   ├── [PREFIX_]Module_1 .. N
│   │   ├── Connector
│   │   └── Type-specific parts
│   └── Interior Objects
│       ├── [PREFIX_]Bridge_Floor
│       ├── [PREFIX_]Corridor_Floor
│       ├── [PREFIX_]Quarters_Floor
│       ├── [PREFIX_]Cargo_Floor
│       ├── [PREFIX_]Engineering_Floor
│       └── ...
```

The `[PREFIX_]` is applied when `naming_prefix` is set (e.g., `NOVAFORGE_Hull`).

### Hardpoint Convention

Hardpoint objects are empties prefixed with `hp_`:
- `hp_engine_0`, `hp_engine_1`, ...
- `hp_weapon_0`, `hp_weapon_1`, ...
- `hp_spine`
- `hp_dock_0`
